# Internationalization

The application renders each page under a language-specific prefix (`/en/...` or `/de/...`). The React app uses [i18next](https://www.i18next.com/) with lazy-loaded JSON resources so only the active language is downloaded.

## Adding or updating translations

1. Add keys to `frontend/src/locales/en/common.json` and `frontend/src/locales/de/common.json`. Use stable, descriptive keys such as `actions.logSighting`.
2. Reference translations in components with `useTranslation` and the `t()` helper.
3. When introducing a new language, create `frontend/src/locales/<lang>/common.json` and update `loadLocale` in `frontend/src/i18n.js` if necessary.
4. Run `npm --prefix frontend run lint` and `pytest -q` to ensure everything passes.

Each route sets the `<html lang>` attribute and provides `<link rel="alternate">` tags for SEO. The language switcher preserves the current path and hash while toggling between locales.
